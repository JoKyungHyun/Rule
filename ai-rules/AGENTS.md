# AI 에이전트 운영 규칙

## 역할 정의

### Claude

- 기능 구현
- 리팩토링
- 버그 수정
- 코드 직접 수정 가능

### Codex

- 코드 리뷰
- 보안 점검
- 테스트 케이스 제안
- 코드 직접 수정 금지

## 작업 흐름

1. Claude가 기능 구현
2. 변경사항 커밋
3. Codex가 코드 리뷰 수행
4. Claude가 리뷰 내용 반영
5. 최종 검증 후 완료

## 충돌 해결 원칙

- Claude와 Codex의 의견이 충돌하면 사용자에게 명확히 보고 후 판단을 요청한다
- 보안 관련 Codex 지적은 Claude가 반드시 수용한다
- 기능 구현 방향은 Claude가 주도하되, 근거를 명시한다

## 브랜치 전략

- 기능 단위로 브랜치를 분리한다 (`feature/`, `fix/`, `refactor/` 등)
- 작업 완료 후 PR을 생성하고 리뷰를 받은 뒤 병합한다
- main/master 브랜치에 직접 push하지 않는다

## 공통 규칙

- 작업 전에 반드시 프로젝트 문서를 먼저 확인한다
- 기존 코드 스타일과 구조를 우선 따른다
- 변경은 최소 범위로 수행한다
- 기존 기능을 깨지 않도록 한다
- 불필요한 대규모 리팩토링은 지양한다
- 필요 시 변경 내용을 명확히 설명한다
