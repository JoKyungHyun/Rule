# 코딩 규칙

## 기본 원칙

- 읽기 쉽고 유지보수 가능한 코드를 작성한다
- 단순한 구조를 우선한다
- 기존 코드 스타일을 따른다

## 네이밍

- 변수명과 함수명은 의미 있게 작성한다
- 불필요한 축약어 사용을 지양한다
- 일관된 네이밍 규칙을 유지한다

## 구조

- 함수는 하나의 역할만 수행하도록 작성한다
- 큰 컴포넌트는 작은 단위로 분리한다
- 중첩된 로직을 최소화한다

## 파일 구성

- import 순서: 외부 라이브러리 → 내부 모듈 → 타입/인터페이스 순으로 정렬한다
- 관련 파일은 같은 디렉토리에 모아둔다
- index 파일을 통해 모듈을 외부에 노출한다

## 타입 관리

- TypeScript 사용을 권장한다
- `any` 타입 사용을 지양한다
- 명확한 타입과 인터페이스를 정의한다
- 공유 타입은 별도 파일(`types/`)에 분리한다

## 에러 처리

- 모든 예외 상황을 고려한다
- 에러를 무시하지 않는다
- 사용자에게 의미 있는 에러 메시지를 제공한다

## 재사용성

- 공통 로직은 함수 또는 컴포넌트로 분리한다
- 코드 중복을 제거한다

## 비동기 처리

- `async/await` 패턴을 사용한다
- Promise 에러를 반드시 처리한다

## 테스트

- 핵심 비즈니스 로직에는 단위 테스트를 작성한다
- 외부 의존성은 mock 처리한다
- 테스트 파일은 대상 파일과 같은 경로에 위치시킨다 (`.test.ts` / `.spec.ts`)

## 린터 / 포매터

- ESLint 규칙을 준수한다
- Prettier 설정을 따른다
- 커밋 전 lint 오류가 없는지 확인한다

## 주석

- 필요한 경우에만 주석을 작성한다
- "무엇"이 아니라 "왜"를 설명한다

## 코드 수정

- 필요한 부분만 수정한다
- 관련 없는 코드 변경을 지양한다
- 변경 사항은 명확하게 유지한다
