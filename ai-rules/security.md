# 보안 규칙

## 입력값 검증

- 모든 사용자 입력값을 검증한다
- XSS, SQL Injection, 명령어 삽입 공격을 방지한다
- 입력값은 반드시 정제 후 처리한다

## 인증 및 권한 관리

- JWT 또는 세션 기반 인증을 사용한다
- 토큰을 클라이언트 코드에 노출하지 않는다
- 최소 권한 원칙을 적용한다
- JWT는 만료 시간(`exp`)을 반드시 설정한다
- Refresh Token은 HttpOnly 쿠키에 저장한다

## 개인정보 보호

- 비밀번호, 주민등록번호 등 민감정보는 로그에 남기지 않는다
- 필요 시 개인정보는 마스킹 처리한다
- 민감정보는 암호화하여 저장한다
- 비밀번호는 bcrypt 등 단방향 해시로 저장한다

## 파일 업로드

- 파일 확장자와 MIME 타입을 검증한다
- 파일 크기 제한을 설정한다
- 악성 파일 여부를 확인한다
- 업로드된 파일은 원본 파일명을 그대로 사용하지 않는다

## 로그 관리

- 사용자 행동 로그를 기록한다 (사용자 ID, 시간 포함)
- 민감정보는 절대 로그에 포함하지 않는다
- 로그는 안전하게 저장한다

## API 보안

- 모든 요청 파라미터를 검증한다
- Rate Limiting을 적용하여 과도한 요청을 차단한다
- CORS는 허용할 출처(origin)를 명시적으로 지정한다 (`*` 사용 금지)
- 모든 API는 HTTPS로만 접근 가능하도록 강제한다

## 의존성 관리

- 주기적으로 `npm audit` 또는 `pip-audit` 등으로 취약점을 점검한다
- 취약점이 발견된 패키지는 즉시 업데이트하거나 대체한다
- 불필요한 의존성은 제거한다

## 보안 취약점 방지

- 비밀키를 코드에 하드코딩하지 않는다
- 내부 에러 메시지를 외부에 노출하지 않는다
- 환경 변수로 민감 정보를 관리한다
- 보안 관련 헤더를 설정한다 (`Content-Security-Policy`, `X-Frame-Options` 등)
